# Краткая пояснительная записка

## 1. Цель проекта

Цель проекта — разработать MVP веб-приложения для проведения интерактивных квизов в реальном времени. Приложение позволяет организатору создавать квизы, запускать комнату по коду, показывать вопросы участникам синхронно и выводить итоговый лидерборд.

## 2. Выбор технологий

Для frontend выбран React, так как он позволяет удобно разбивать интерфейс на компоненты: авторизация, кабинет, редактор квиза, комната организатора, экран участника и лидерборд. Для сборки используется Vite, потому что он быстро запускает проект и подходит для MVP.

Для backend выбран Node.js и Express. Такой стек хорошо подходит для JavaScript-проекта и позволяет быстро реализовать API для регистрации, создания квизов и хранения истории.

Для взаимодействия в реальном времени используется Socket.IO. Он позволяет синхронно отправлять события всем участникам комнаты: запуск вопроса, отправка ответа, обновление баллов и завершение квиза.

Для хранения данных в MVP используется JSON-файл на сервере. В промышленной версии его можно заменить на PostgreSQL или MongoDB.

## 3. Основные этапы разработки

1. Проектирование пользовательских сценариев для организатора и участника.
2. Создание модели данных: пользователи, квизы, вопросы, комнаты, история результатов.
3. Разработка API для регистрации, авторизации, создания и запуска квизов.
4. Разработка интерфейса на React.
5. Настройка Socket.IO для работы комнат в реальном времени.
6. Тестирование сценария: организатор запускает квиз, участник подключается по коду, отвечает на вопросы, система считает баллы.

## 4. Реализованная функциональность

В MVP реализованы регистрация и вход пользователей, разделение ролей на организатора и участника, создание квизов, добавление вопросов с одиночным и множественным выбором ответа, запуск комнаты по коду, подключение участников, синхронный показ вопросов, отправка ответов, подсчёт баллов, лидерборд и сохранение истории проведённых квизов.

## 5. Вывод

Проект показывает основную бизнес-логику платформы для интерактивных квизов. MVP можно использовать как основу для дальнейшего развития: добавить полноценную базу данных, загрузку изображений, расширенную аналитику, таймер на сервере, админ-панель и деплой в production.
